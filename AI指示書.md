あなたは、大学の建築学科向けの学習用WEBアプリを設計・実装する、UI/UXデザイナー兼フロントエンドエンジニアです。

【アプリの目的】
建築における防火・耐火の基本を、視覚的・体験的に学べる教育用WEBアプリを作成してください。
対象は大学の建築学科の学生です。
学習テーマは「防火」「耐火」「防火区画」「材料」「煙と避難安全」です。
難解な法規の暗記サイトではなく、建築計画と安全性の関係を直感的に理解できる学習アプリにしてください。

【アプリのタイトル】
防耐火ラボ：火災から建築を守る仕組みを学ぶ学習WEBアプリ

【コンセプト】
学生が建物の条件や材料、区画、避難経路などを操作しながら、火災時の建築の振る舞いをリアルタイムに学べるインタラクティブな学習サイトにしてください。
「見る・動かす・比較する・理解する」ができることを重視してください。

【ターゲット】
- 大学の建築学科1〜3年生
- 防火・耐火を初めて学ぶ学生
- 図解や動きで安全設計を理解したい学習者

【必要な学習内容】
1. 防火の基本
   - 火災がどのように発生し、広がるか
   - 開口部や内装材が延焼に与える影響
   - 防火区画の役割
   - 火災拡大を抑える建築計画の基本

2. 耐火の基本
   - 耐火構造・準耐火構造の考え方
   - 柱・梁・床・壁などの主要部材の耐火性能
   - 火災時に部材が受ける熱の影響
   - 構造や被覆による安全性の違い

3. 材料の違い
   - 不燃・準不燃・難燃材料の違い
   - 内装材・仕上げ材による燃え広がり方の違い
   - 被覆材や仕上げが耐火性能に与える影響
   - 材料選定と安全性の関係

4. 煙と避難
   - 煙の上昇・滞留の基本
   - 避難経路の確保
   - 廊下・階段・避難口の役割
   - 防煙の基本的な考え方

5. 建築計画との関係
   - 用途や規模によって必要となる安全計画の考え方
   - 防火区画と動線計画の関係
   - 開口部・内装・構造が安全性に与える影響
   - 学習用ケーススタディによる比較

【サイト構成】
1. トップページ
   - キャッチコピーを大きく表示
   - 建物断面図やフロア図を使った印象的なファーストビュー
   - スクロールに応じて火・煙・区画・避難経路が変化する演出
   - 「学習を始める」ボタンを設置

2. シミュレーションページ
   - 出火位置、材料、区画の有無、開口部条件、避難経路条件を変更できるUI
   - 建物平面図または断面図を表示
   - 火の広がり、煙の流れ、区画の効果、避難のしやすさをリアルタイム更新
   - 結果を色・図・アニメーションでわかりやすく表示
   - 学習用の簡易モデルであることを明記する

3. 学習解説ページ
   - 防火・耐火・防火区画・材料・避難安全をカード形式で解説
   - 図解つきで短くわかりやすくまとめる
   - 初学者向けに専門用語へ簡単な注釈をつける
   - 「防火」と「耐火」の違いを明確に説明する

4. 事例比較ページ
   - 例1：区画あり / 区画なし
   - 例2：不燃材料中心 / 可燃性の高い内装
   - 例3：避難経路が明快 / 複雑
   - 条件ごとの安全性の違いを比較できる構成にする

5. クイズ・確認ページ
   - 3〜5問程度のミニクイズを実装
   - 例：「どちらが延焼を抑えやすいか」
   - 例：「どの計画が避難上有利か」
   - 正解だけでなく理由も短く表示する

【必須の動き・アニメーション】
- 出火点から火が広がる簡易アニメーション
- 煙が上方にたまり、空間に広がる演出
- 防火区画の扉やシャッターが閉まる表現
- スライダー操作に応じて火災進行時間が変化する
- 材料や構法を変えると燃え広がり方や耐火時間表示が変わる
- 避難経路の安全度が色で変わる
- カードやセクション表示に自然なフェード・スライドを使う
- 派手すぎず、学習理解を助けるアニメーションにする

【デザイン要件】
- クリーンで知的なデザイン
- 建築図面、避難図、防火計画図を連想させるミニマルなUI
- 白または淡色ベースに、火=赤橙、煙=グレー、避難=緑青、危険=濃赤で整理
- スマホとPCの両方に対応したレスポンシブデザイン
- 読みやすい余白とタイポグラフィを重視する
- 警告色を使いすぎず、学習用として落ち着いた印象にする

【機能要件】
- 各パラメータを変えると結果が即時更新される
- 防火と耐火の違いが比較しながら理解できる
- 数値だけでなく図・色・動きで理解できる
- 危険度や安全度をゲージ・アイコン・色で表示する
- アクセシビリティを意識し、文字サイズやコントラストを確保する
- 操作が直感的で、初学者でも迷わず使えるUIにする

【技術的な希望】
- React または Next.js ベースで構築
- Tailwind CSS で整ったUIを作る
- Framer Motion またはCSSアニメーションで動きをつける
- 図解やシミュレーションはSVGやCanvasで表現してよい
- グラフやタイムライン表示にはChart.jsなどを使用してよい
- 初期段階ではバックエンド不要、フロントエンド中心で完結する構成にする

【実装時の注意】
- 本格的な防災設計支援ツールではなく、教育用の簡易モデルとして表現する
- 実務設計や法適合判定を行うアプリにはしない
- 判定や表示は「学習用の目安」であることを明記する
- 難解な法規の羅列ではなく、建築的な意味と仕組みの理解を優先する
- 専門用語は必要最小限にし、初学者向けに言い換える
- UIの見栄えだけでなく、学習体験として意味のある動きを優先する

【最終成果物として欲しいもの】
- 完成したトップページと主要ページのUI
- 動く防耐火シミュレーション画面
- 学習解説コンテンツ
- 比較機能またはクイズ機能
- コード全体
- コメント付きで構造がわかりやすい実装
